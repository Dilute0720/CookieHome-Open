# CHANGELOG

## 2026-06-16

### Added

- `TimelineEvent` 新增 `sourceBlogPostId` 来源字段，同一条小记只能设为一条时间轴重要记忆。
- 新增 `npm run inventory:import` 家庭库存批量入库 CLI，支持 JSON 文件/stdin、dry-run 预览和机器可读输出，便于 AI 生成入库信息。
- 新增 `docs/INVENTORY_IMPORT.md`，记录库存批量入库 JSON 格式和给 AI 的输出提示词。

### Changed

- 重构博客和时间轴的信息架构：博客定位为日常流，时间轴定位为精选档案。
- 博客小记操作从“加入时间轴”调整为“设为重要记忆”，已沉淀的小记会显示时间轴收藏状态。
- 时间轴手动新增改为折叠的补录入口，侧栏改为“待整理的小记”，只展示尚未沉淀为重要记忆的候选小记。

## 2026-06-13

### Added

- 点餐页新增移动端已选菜品结算条，实时展示已选数量和菜名，并支持点选菜名快速取消。
- 库存页新增搜索和状态筛选，可按食材/关联菜谱搜索，并快速查看全部、临期、已过期和可做菜库存。
- 点餐页显示当前计划日期下“我已点 / 家里已点”的菜品状态，并在切换日期后同步刷新。
- 点餐页新增菜品搜索和“全部 / 未点 / 我已点 / 家里已点”筛选，方便手机端快速追加点餐。
- 库存页新增“处理优先 / 名称 / 批次多 / 可做菜”排序，默认把更需要处理的临期库存排在前面。
- 库存批次扣减支持填写实际扣减数量。

## 2026-06-06

### Added

- 曲奇堡日记支持图文小记首版：发布时可上传照片并填写标签，列表中展示图片网格和标签。
- 曲奇堡日记的小记可一键沉淀为家庭时间轴事件，保留原小记作者和创建日期。
- 待办菜单支持针对单道菜标记烹饪完成，完成后该菜会从待办菜单清除并保留历史记录。
- 曲奇堡日记“最近的小动静”支持撤回菜单完成记录，可把整单或单菜重新放回待办菜单。
- 新增公开展示页 `/family-os`，发布 Family OS 苹果发布会风格单页。

### Changed

- 点餐提交改为追加到待办菜单，不再覆盖同一成员同一日期已有点餐；追加新菜会把已完成菜单重置为待处理。
- 整单烹饪完成后会把当前待办菜品标记完成并从待办菜单清除，历史数据继续用于菜品最近制作记录和后续统计。
- 网页 app logo 替换为曲奇堡插画图标，并同步更新 favicon、Apple 主屏幕图标、PWA manifest 图标和顶部导航品牌图标。
- 待办菜单状态文案调整为“待处理 / 采购完成 / 烹饪完成”，并允许任意已登录家庭成员更新进度。
- 家庭时间轴照片编辑支持上移、下移和移除，保存后照片顺序会写回 `TimelineEvent.images`。
- 待办菜单页移除前台点餐统计，右侧改为展示按库存抵扣后的食材缺口，并提供买菜清单入口。

### Fixed

- 修复菜品编辑时单位输入框清空后又恢复默认单位的问题。

## 2026-06-05

### Added

- 新增家庭时间轴 `/timeline` 首版：支持记录家庭事件，上传照片，按年份和月份展示，作者或管理员可编辑/删除。

### Changed

- 家庭时间轴照片复用当前本地上传、服务端压缩和 `/uploads` 兜底读取链路。
- 增强移动端基础体验：新增底部高频导航、PWA manifest、生成式 app 图标、手机安全区留白和触控优化。
- 待办菜单标记烹饪完成时会按菜谱食材自动扣减库存，并在家庭动态记录库存扣减和不足摘要。
- 待办菜单状态卡新增本次实际用量确认，完成前可调整扣库存数量。
- 待办菜单完成后新增每道菜的快速做菜日志入口，便于把实际做法沉淀回菜谱成长时间线。
- 买菜清单勾选状态持久化到数据库，按计划日期保留已买状态，并在快捷入库后自动标记为已买。
- 库存临期区新增处理入口：可直接标记用完、丢弃单个批次或一键清理所有已过期批次，首页同步显示临期库存提醒。
- 细化家庭角色权限：菜品管理、库存写入、用户管理和快捷入库仅管理员可操作，Family 保留日常点餐、待办菜单进度更新、浏览、博客和清单勾选。
- 字体改用系统中文字体栈，移除 Google Fonts 构建/运行时依赖。
- “明日菜单”前台语义调整为“待办菜单”，默认仍取明天，但点餐、推荐采纳、待办菜单和买菜清单支持选择计划日期。
- 买菜清单增加勾选采购项快捷入库，入库前可调整实际数量。
- 买菜清单、库存汇总、菜品详情库存满足度和推荐逻辑支持重量/体积单位换算，修复 `mL`、`L` 等单位后端匹配分裂的问题。
- 食材和库存单位输入升级为可搜索、可创建的分组单位选择器，并增加常见单位别名归一。
- 清除登录页账号输入框中的默认账号提示词。
- 菜品封面上传改为服务端转码：保存前自动旋转、压缩并统一输出为 JPEG，提升手机照片上传后的显示稳定性。
- 菜品封面上传入口支持 20MB 内手机原图，超限会在前端提示，最终保存文件仍会压缩落盘。

### Fixed

- 登录保护放行 PWA manifest 和 app icon 路径，避免手机浏览器无法读取安装信息。
- 修复退出登录和登录保护重定向在反向代理部署下可能跳回 `localhost` 的问题。
- 修复中文文件名上传图片路径未编码导致 `香蕉苹果奶昔.png` 等图片可能无法显示的问题。
- 修复上传图片可能被登录中间件拦截或因旧路径格式无法解析导致显示失败的问题；本地上传 URL 统一规范为 `/uploads/...`。
- 修复移动端上传 HEIC/HEIF 或异常图片时可能被保存为浏览器不可显示文件的问题；不支持解码时会在表单内给出明确提示。
- 修复生产部署中运行时新增上传图片存在于 `public/uploads` 但访问 `/uploads/...` 返回 404 的问题。
- 修复移动端顶部菜单展开后点击页面空白不会自动关闭的问题。
- 修复移动端菜谱保存上传原图时可能因 Server Action 默认 1MB 请求体限制而出现通用服务端错误的问题。
- 补充库存写入相关 Server Action 登录校验，避免直接调用库存变更。

## 2026-06-04

### Added

- 小记支持重编辑，评论区支持按楼层展示和追加评论；发布小记后会自动刷新回日记页。
- 小记删除功能：作者或管理员可通过更多菜单删除用户主动发布的小记，删除前需要确认。
- 菜品标签输入支持自定义标签和历史标签建议，保存时会复用已有规范标签。
- 库存系统新增批次管理：支持入库日期、保质期天数、失效日期和备注。
- 库存页新增库存汇总、批次明细和临期/过期状态提示。
- 新增成员账号登录：登录会话携带当前用户身份，支持按账号密码进入小家。
- 新增管理员用户后台 `/admin/users`，支持新增成员、修改账号资料和重置密码。
- 明日菜单新增做饭人选择，菜单完成时自动写入博客自动记录。
- 博客页新增留言发布和“全部 / 留言 / 自动记录”筛选。
- `User` 新增 `username` 和 `passwordHash` 字段，`TomorrowMenu` 新增做饭人和完成时间，`BlogPost` 新增内容类型。

### Changed

- 博客页重构为“曲奇堡日记”：主内容流只展示用户主动发布的小记，系统流水账弱化为“最近的小动静”侧栏，并更新页面文案和移动端布局。
- 库存页删除无意义的跨单位总量统计，临期区域前置并强化视觉优先级，统计区改为紧凑概览条；顶部导航按家庭使用场景重排并支持当前页高亮和移动端折叠菜单。
- 菜品详情页重构为移动端优先的菜谱页面：无封面时隐藏图片区域，增加家庭评分、库存满足度、“今晚做这个”、紧凑食材库存状态、成长/心得时间轴、库存联动提示和最近制作记录。
- 库存页面改为移动端优先卡片列表，新增库存改用弹窗，批次信息和编辑表单默认折叠，并增加临期区、统计和菜谱关联推荐。
- 菜品食材编辑器改为紧凑列表模式，点击单行展开编辑，并在行内展示库存状态。
- 菜品编辑页食材区域改为主料/配料分区，每行压缩为名称、数量、单位和删除操作。
- 菜品库标签筛选会展示历史自定义标签，不再只展示内置标签。
- 买菜清单和推荐逻辑会按同名同单位汇总多个库存批次数量后再抵扣。
- 点餐页和推荐页改为使用当前登录用户作为点餐来源，不再手动选择用户。
- 默认种子用户调整为 `me`、`sister`、`yuyuan` 三个账号。

## 2026-06-03

### Added

- 预留 OSS 图片存储 driver 和环境变量，当前默认仍使用本地 `public/uploads`。
- 新增 `docs/DEPLOYMENT.md`，用于指导 ChatGPT 协助完成 VPS、SQLite、PM2、Nginx、HTTPS、备份和更新部署流程。
- 新增家庭密码登录门禁：`/login` 登录页、httpOnly cookie 会话、`proxy.ts` 全站保护和退出入口。
- `.env.example` 新增 `FAMILY_ACCESS_PASSWORD` 和 `FAMILY_AUTH_TOKEN` 配置。
- 添加登录跳转安全归一化测试。

### Fixed

- 关闭顶部导航的 Next.js 预取，避免登录页未认证状态预取到登录跳转后，登录成功仍复用旧跳转结果。

- 修复 HTTP 直连生产服务时登录 cookie 因 Secure 标记无法发送，导致登录后反复回到登录页的问题。

### Changed

- 菜品和库存的食材名称输入改为类 Obsidian 标签体验：自动提示常用食材、历史菜谱食材和库存名称，保存时优先复用已有规范名称。
- 菜谱食材输入从“预设下拉 + 自定义框”改为直接输入食材名称，并通过输入建议保留常用食材选择。
- 将文件存储预留说明从 S3 兼容对象存储调整为 OSS 预留接口，避免部署阶段误切换存储 driver。
