# ROADMAP

## 当前状态

Phase 1 已完成：

- 菜品库
- 菜品详情页
- 点餐系统
- 明日菜单
- 买菜清单
- 基础用户体系

系统已经具备可用 MVP。

后续开发遵循以下路线。

## Phase 2：库存管理系统（最高优先级）

目标：让买菜清单真正具备实用价值。

### 功能目标

新增家庭库存 `Inventory`，例如：

- 鸡蛋 8个
- 番茄 3个
- 牛奶 2盒
- 可乐 1瓶

### 库存管理页面

路径：`/inventory`

支持：

- 新增库存
- 修改库存
- 扣减库存
- 删除库存

### 菜品与库存联动

当生成明日菜单时，系统自动计算：

```text
需要食材 - 当前库存 = 缺少食材
```

生成最终采购清单。

### 数据模型

```text
Inventory {
  id
  name
  quantity
  unit
  stockedAt
  shelfLifeDays
  expiresAt
  note
  updatedAt
}
```

## Phase 3：菜谱成长系统

目标：记录做菜经验积累。

### 功能目标

每个菜品支持做菜日志，例如：

- 2026-06-01：第一次尝试
- 2026-06-15：减少盐
- 2026-07-10：加入八角
- 2026-08-01：最终版本

### CookingJournal

```text
CookingJournal {
  id
  dishId
  content
  createdAt
}
```

### 菜谱版本历史

展示最佳做法演进过程，形成真正的家庭食谱库。

## Phase 4：智能菜单推荐

目标：解决“今天吃什么”。

### 随机推荐

新增按钮：`帮我决定`

随机推荐菜单。

### 智能推荐

结合：

- 库存
- 最近菜单
- 菜品热度
- 季节

推荐今日菜单。

### 推荐规则

避免：

- 连续三天红烧肉
- 连续多天重复菜单

提高多样性。

## Phase 5：家庭时间轴

目标：构建家庭记忆系统。

当前进度：首版已完成，支持家庭事件记录、年份/月分组展示、作者/管理员编辑删除、照片上传、照片移除与排序；已重构为精选档案，支持从博客小记设为重要记忆，并记录来源关系。

路径：`/timeline`

### 时间轴

展示：

- 2026
- 6月：第一次上线网站
- 7月：长沙旅行
- 8月：学会红烧排骨
- 9月：姐姐生日

### TimelineEvent

```text
TimelineEvent {
  id
  title
  content
  date
  images
}
```

## Phase 6：博客系统

目标：记录家庭生活。

当前进度：首版已完成，支持图文小记、图片上传、标签展示、评论；已明确为日常流，用户可把值得长期保存的小记设为时间轴重要记忆。

路径：`/blog`

支持：

- 图文发布
- 图片上传
- 评论
- 标签

不需要：

- 点赞
- 转发
- 关注
- 推荐算法

重点：家庭记录。

## Phase 7：家庭统计中心

目标：沉淀长期数据价值。

### 菜品统计

展示：

- 最受欢迎菜品
- 做得最多菜品
- 最长未制作菜品

### 用户统计

展示：

- 谁点菜最多
- 谁最爱吃什么

### 做饭统计

展示：

- 年度做饭次数
- 年度新增菜品数
- 年度采购次数

## Phase 9：家庭 AI 助手

目标：成为家庭生活助手。

### 首页今日建议

例如：

```text
今天较热。

库存：
- 鸡蛋
- 黄瓜
- 番茄

推荐：
- 番茄炒蛋
- 凉拌黄瓜
```

### AI 能力

支持：

- 菜单推荐：今天吃什么
- 买菜建议：缺少什么
- 菜谱问答：上次红烧肉怎么做最好吃？
- 家庭数据总结：最近一个月最常做的菜

## 开发原则

优先级：

```text
库存管理
＞
菜谱成长
＞
智能推荐
＞
时间轴
＞
博客
＞
统计分析
＞
AI
```

保持：

- 先交付可用功能
- 再做高级功能
- 避免过早设计复杂系统
